# 贷款真相计算器

## 概述

中国贷款市场充斥着各种包装术语——"月费率0.3%"、"日息万五"、"手续费率0.6%"、"先息后本到期还本"……这些说法让用户误以为利率很低，但实际年化利率可能是宣传数字的1.8~2.4倍。

本技能通过精确计算 **IRR（内部收益率）**，将所有贷款方案还原为统一标准的**真实年化利率（APR，复利公式 `(1+r)^12-1`）**，让用户看清每个产品的真实成本。支持单方案分析和多方案对比。

**核心能力**：
- 精确计算各种还款方式下的真实年化利率
- 识别常见贷款套路并给出风险提示
- 多方案横向对比，按真实成本排序
- 生成逐期还款计划表

## 触发条件

当用户提到以下内容时触发此技能：
- 计算贷款利息/利率/年化利率/APR
- 对比多个贷款方案
- 不理解某个贷款术语（月费率、日息、手续费率、先息后本等）
- 怀疑贷款利率被包装了
- 输入包含贷款金额、期数、还款金额、利率/费率等关键词

## 使用方法

### 输入格式

用户可以用自然语言描述贷款信息，技能自动解析。最少需要以下信息：

**必需**：贷款金额、期数（月）、还款方式或每期还款额
**可选**：手续费/服务费/担保费等额外费用、日利率/月费率

**示例输入**：
- "借呗借1万，分12期，每期还880"
- "银行月费率0.28%，借5万3年"
- "对比A方案月费率0.3%分12期和B方案日息万四随借随还"
- "借5万，A是月费率0.3%分12期，B是日息万四"（无需"方案"二字也能识别）
- "网贷借1万扣200服务费，分24期每期500"
- "借5000到手只有4000，分12期每月还500"（砍头息+真实月供，优先用真实月供反推）

### 解析流程

1. 从用户输入中提取：产品名称、贷款金额、期数、还款方式、每期还款额、费率/利率、额外费用
2. 识别贷款类型和计息方式
3. 调用计算器计算真实 APR
4. 生成报告

## 支持的还款方式

### 1. 等额本息
每月还款金额相同。最常见的还款方式。
- 银行信用贷、消费贷常用
- 宣传"月费率0.3%"时，实际年化 ≈ 6.58%（月费率 × 12 × 1.8 ≈ 6.48%）

### 2. 等额本金
每月归还本金相同，利息逐月递减。首月还款最多，逐月递减。
- 房贷常用
- 总利息比等额本息少

### 3. 先息后本
每月只还利息，到期一次性还本金。
- 经营贷、部分信用贷
- 表面利率很低，但到期需一次性拿出大额本金

### 4. 按期付息到期还本
同先息后本。

### 5. 随借随还（按日计息）
按实际占用天数计息，随时可还。
- 借呗、微粒贷、京东金条
- 日息万五 = 年化18.25%

### 6. 信用卡分期
按月收取手续费，手续费按全额收取（不按剩余本金）。
- 实际年化 ≈ 手续费率 × 12 × 1.8

### 7. 砍头息
服务费/管理费在放款前预先扣除。
- 实际到手 < 合同金额
- 真实成本远高于合同利率

### 8. 气球贷
前期小额还款，最后一期大额尾款。
- 前期压力小，最后一期集中还款

## 计算公式

### IRR（内部收益率）
使用 Newton-Raphson 迭代法求解：

```
NPV = Σ(CF_i / (1 + r)^i) = 0
```

其中 CF_i 为第 i 期的现金流，r 为单期利率。

真实年化利率 APR = (1+r)^12 - 1（月付，复利年化）或 APR = (1+r)^365 - 1（日付，复利年化）

### 等额本息月供
```
月供 = P × r × (1+r)^n / ((1+r)^n - 1)
```

### 月费率 → 真实年化
```
IRR 求解：-P + Σ[(P/n + P×月费率)/(1+r)^i] = 0
APR = (1+r)^12 - 1  （复利年化，符合监管标准）
近似：APR ≈ 月费率 × 12 × 1.8
```

## 输出格式

### 单方案报告
1. **简洁结果**：真实年化利率（大字突出）
2. **对比信息**：宣传利率 vs 真实利率
3. **总成本**：总还款额、总利息
4. **风险提示**：识别可能的套路
5. **还款计划表**：逐期本金、利息、剩余本金

### 多方案对比
1. **排序表格**：按真实 APR 从低到高排列
2. **最优方案**：推荐最划算的方案
3. **成本差异**：方案间利息差额

## 执行方式（必须严格遵守）

⚠️ **重要：你必须执行 Python 脚本，不能用模型自己算！**

### 执行步骤

1. **解析输入**：从用户自然语言中提取贷款参数（金额、期数、费率等）
2. **执行脚本**：在 `scripts/` 目录下运行 Python 脚本
   ```bash
   cd scripts/
   python3 -c "
   import sys
   sys.path.insert(0, '.')
   from parser import parse_loan_input
   from report import generate_report
   
   text = '用户输入的自然语言'
   parsed = parse_loan_input(text)
   report = generate_report(parsed['products'], parsed['compare'])
   print(report)
   "
   ```
3. **输出报告**：将脚本输出的报告展示给用户

### 为什么必须用脚本？

- 模型直接计算 IRR 精度差，经常算错
- 模型对"月费率"和"年化利率"的关系理解模糊
- 脚本使用 Newton-Raphson 精确求解，小数点后两位都准
- 脚本支持复利 APR = (1+irr)^12 - 1，符合监管标准

### 错误示范 ❌

```
❌ 直接用模型推理："月费率0.3%大概是年化6%左右"
❌ 用模型手算："10000元分12期，每期880，利息560..."
❌ 跳过脚本，自己估算
❌ 脚本返回多个利率字段时，自己判断该用哪个、该怎么换算
   （例如同时看到 monthly_fee_rate 和 stated_apr 时，自己去"理解"两者关系）
```

### 正确示范 ✅

```
✅ 执行 python3 scripts/calculator.py 调用 calc_credit_card_installment()
✅ 执行 python3 scripts/report.py 生成完整报告
✅ 严格按照脚本输出展示给用户，一字不改地转述报告中的数字和结论
```

### 字段歧义处理规则（重要）

parser 解析结果中可能同时存在多个利率字段，**禁止自行判断该用哪个或如何换算**，
必须严格按以下优先级，且优先级本身已经在 report.py 中处理好，
你只需要原样输出 `generate_report()` 的返回内容：

- `stated_apr`：用户转述的"销售/宣传/广告说的"年化，**只用于报告中的对比展示**，不参与计算
- `monthly_fee_rate` / `daily_rate` / `monthly_payment`：计费基础，参与实际计算
- 若同时出现 `stated_apr` 和其他计费字段，报告里"宣传值 vs 真实值"的对比已经由脚本算好，
  **不要自己再算一遍倍数或差值**，直接转述脚本输出的句子

## 注意事项

- 所有计算基于中国银保监会规定的 IRR 方法
- 真实年化利率 = (1+单期利率)^12 - 1（按月，复利）或 (1+单期利率)^365 - 1（按日，复利），符合央行〔2021〕3号公告
- 实际利率可能因具体合同条款略有差异，仅供参考
