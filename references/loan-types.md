# 各类贷款产品计息方式详解

## 1. 等额本息（Equal Principal and Interest）

**最常见**的还款方式，每月还款金额相同。

### 公式
```
月供 = P × r × (1+r)^n / ((1+r)^n - 1)
```
- P = 贷款本金
- r = 月利率（年利率/12）
- n = 期数（月）

### 特点
- 每月还款额固定，便于规划
- 前期利息占比高，后期本金占比高
- 总利息比等额本金多

### 适用产品
- 银行消费贷
- 大部分信用卡分期
- 房贷（主流选择）

---

## 2. 等额本金（Equal Principal）

每月归还本金相同，利息逐月递减。

### 公式
```
每月本金 = P / n
每月利息 = 剩余本金 × r
首月还款 = P/n + P×r
逐月递减 = P/n × r
```

### 特点
- 首月还款最多，逐月递减
- 总利息比等额本息少
- 前期还款压力大

### 适用产品
- 房贷（常见选择）
- 部分消费贷

---

## 3. 先息后本（Interest First, Principal Last）

每月只还利息，到期一次性还本金。

### 公式
```
每月利息 = P × r
到期本金 = P
总利息 = P × r × n
```

### 特点
- 表面利率极低（只看月利息）
- 到期需一次性拿出大额本金
- 如果续贷会产生额外费用

### 适用产品
- 经营贷
- 部分信用贷
- 过桥贷款

---

## 4. 信用卡分期（Credit Card Installment）

**最经典的包装利率套路。**

### 计算方式
```
每月手续费 = 本金 × 月费率（按全额计算）
每月本金 = 本金 / 期数
每月还款 = 每月本金 + 每月手续费
```

### 关键陷阱
手续费按**借款全额**计算，不按剩余本金递减。
你第12个月只剩几百元本金，还在交和第一个月一样的手续费！

### 真实年化公式
```
IRR 求解: -P + Σ[(P/n + P×月费率)/(1+r)^i] = 0
APR = (1+r)^12 - 1  （复利年化）
近似公式: APR ≈ 月费率 × 12 × 1.8
```

### 常见月费率（精确值，12期）

| 月费率 | 宣传年化 | 真实APR | 倍数 |
|--------|---------|---------|------|
| 0.25% | 3.00% | 5.63% | ×1.9 |
| 0.28% | 3.36% | 6.32% | ×1.9 |
| 0.30% | 3.60% | 6.78% | ×1.9 |
| 0.40% | 4.80% | 9.10% | ×1.9 |
| 0.50% | 6.00% | 11.46% | ×1.9 |
| 0.60% | 7.20% | 13.84% | ×1.9 |

---

## 5. 网贷日息（Daily Interest）

借呗、微粒贷、京东金条等。

### 计算方式
```
日利率 = 万X（如万五 = 0.0005）
真实年化 = 日利率 × 365
利息 = 本金 × 日利率 × 实际天数
```

### 关键陷阱
- 日息看起来很低（万三、万五），但年化很高
- 随借随还按实际天数计息，提前还款也要付当天利息

### 常见日利率

| 日利率 | 宣传方式 | 真实年化 |
|--------|---------|---------|
| 万三 | 日息万三 | 10.95% |
| 万四 | 日息万四 | 14.60% |
| 万五 | 日息万五 | 18.25% |
| 万一 | 日息万一 | 3.65% |

---

## 6. 砍头息（Head-Chopping Fee）

**最恶劣的贷款套路之一。**

### 计算方式
```
合同金额 = 10000元
服务费 = 合同金额 × 服务费比例（如2%）
实际到手 = 合同金额 - 服务费 = 9800元
每月还款按合同金额计算
```

### 关键陷阱
- 你借1万，实际只拿到9800元
- 但利息按10000元算
- 等于白付了2%的"空气利息"
- 真实利率 = 合同利率 + 服务费折算利率

### 示例
```
借款10000元，服务费2%，月费率0.5%，12期
实际到手：9800元
每月还款：10000/12 + 10000×0.005 = 883.33元
总还款：10600元
真实多付：10600 - 9800 = 800元
真实年化 ≈ 14.74%（而非宣传的6%）
```

### 两种场景，计算路径不同（重要）

**场景A：用户没说每月还多少，只知道服务费率和月费率**
→ 按假定费率算出理论月供（如上面的示例），用 `calc_head_chopping_loan`。

**场景B：用户直接说了"每月还XX元"（最常见的真实场景）**
→ **必须用用户给出的真实月供反推APR，不能套用任何假定费率重新计算月供**。
用户说的月供是事实，不是待求的未知数，用假定值替换会丢弃用户提供的真实信息。

反推时，IRR现金流的起点应该是**实际到手金额**（不是合同金额），
因为用户真实占用的资金就是到手的那部分：

```
现金流 = [-实际到手金额, 月供, 月供, ..., 月供]  （n期）
真实APR = (1+IRR)^12 - 1
```

举例对比：借5000到手4000，分12期每月还500
- 错误做法：套用默认月费率0.3%，重新算出一个"理论月供"（结果会偏离用户说的500）
- 正确做法：直接用现金流 `[-4000, 500, 500, ..., 500]`（12个500）求IRR
- 正确结果：真实年化 ≈ 121.84%（远高于"用假定费率重算"得出的虚假偏低值）

## 7. 气球贷（Balloon Loan）

前期小额还款，最后一期大额尾款。

### 正确计算方式

**贷款本金 = 月供的年金现值 + 尾款的现值**

```
尾款金额 = 本金 × 尾款比例（常见30%-50%）

月供反解公式：
本金 = 月供 × [1-(1+r)^-n]/r + 尾款 × (1+r)^-n

即：月供 = (本金 - 尾款的现值) / [1-(1+r)^-n]/r
```

⚠️ **常见误区（已被证明错误，不要这样算）**：
把本金人为拆成"摊销部分"和"尾款部分"分别独立计算——
即先算 `摊销本金 = 本金 - 尾款`，再单独对这部分本金做等额本息。
这种算法等价于把一笔贷款错误拆成两笔互不相关的贷款，
会导致反推出的真实APR远低于实际年利率（实测偏差可达40%以上），
原因是整个贷款期内本金一直是"全额本金"在计息，
不能简单地把尾款部分从计息基数里提前剔除。

### 示例

```
本金10万元，年利率6%，24期，尾款比例30%
尾款金额 = 100000 × 0.3 = 30000元

月利率 r = 0.06/12 = 0.005
月供 = (100000 - 30000×(1.005)^-24) / [(1-(1.005)^-24)/0.005]
     ≈ 3252.44元

总还款 = 3252.44×24 + 30000 = 108058.56元
真实年化（IRR反推）≈ 6.17%（应接近设定的6%年利率，因为没有额外费用）
```

### 关键陷阱
- 前期看着还款压力很小
- 最后一期要拿出一大笔钱
- 如果拿不出，只能续贷（产生新费用）

## 8. 综合资金成本

很多贷款除了利息还有额外费用：
- 服务费
- 担保费
- 保险费
- 账户管理费
- 评估费

**真实年化利率 = (利息 + 所有费用) 的 IRR**

中国银保监会要求所有贷款产品必须披露 APR（年化综合资金成本），包含所有费用。

---

## 附录：监管规定

### 中国贷款利率法律上限
| 区间 | 法律效力 |
|------|---------|
| ≤ 24% | 完全受法律保护 |
| 24%-36% | 已支付的不退还，未支付的可拒绝 |
| > 36% | 超过部分无效，已支付的可要求返还 |

### 监管要求
- 所有贷款产品必须以 APR（年化综合资金成本）形式披露真实利率
- APR 计算必须包含利息、手续费、担保费等所有费用
- 不得以"费率""服务费"等名义掩盖真实利率
